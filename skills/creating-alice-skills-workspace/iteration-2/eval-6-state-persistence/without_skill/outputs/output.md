# Хранение состояния для викторины в навыке Алисы

Для викторины лучше разделить состояние на три независимых уровня: состояние текущей сессии, долговременный прогресс пользователя и настройки конкретного устройства. Не стоит хранить все в одном объекте: у этих уровней разный срок жизни, область видимости и правила очистки.

## 1. Счет текущей сессии

Используйте поле ответа `session_state`.

Это состояние вернется в следующем запросе в `state.session` и живет только в рамках текущего диалога. Оно подходит для временных данных раунда:

```json
{
  "session_state": {
    "quiz_id": "capitals",
    "question_index": 4,
    "current_score": 3,
    "asked_question_ids": ["q1", "q5", "q9"],
    "last_question_id": "q9"
  }
}
```

Что хранить:

- `quiz_id` - выбранная тема или набор вопросов.
- `question_index` - номер текущего вопроса.
- `current_score` - счет только в этой игровой сессии.
- `asked_question_ids` - вопросы, которые уже были заданы в этой сессии.
- `last_question_id` - вопрос, на который сейчас ожидается ответ.

Как очищать:

- Когда викторина закончилась, передайте пустой `session_state`, например `{}`.
- Если нужно сбросить только часть временных данных, верните `session_state` без этих ключей или с новыми начальными значениями. Для сессионного состояния проще считать ответ новым снимком состояния на следующий ход.

## 2. Общий прогресс авторизованного пользователя между сессиями

Используйте поле ответа `user_state_update`.

Это состояние возвращается в следующих запросах в `state.user` и подходит для прогресса, который должен сохраняться между сессиями для одного и того же пользователя:

```json
{
  "user_state_update": {
    "total_games": 18,
    "total_correct": 126,
    "best_score": 9,
    "completed_quizzes": ["capitals", "history-basic"],
    "level": 3
  }
}
```

Что хранить:

- `total_games` - сколько викторин завершено.
- `total_correct` - общее число правильных ответов.
- `best_score` - лучший результат за одну сессию.
- `completed_quizzes` - завершенные наборы вопросов.
- `level` или `achievements` - долгосрочный прогресс.

Обновляйте это состояние только в устойчивых точках: после завершения игры, выдачи достижения или смены уровня. Не нужно писать туда каждый промежуточный ответ, если эти данные нужны только до конца текущего диалога.

Как очищать:

- Чтобы удалить отдельный ключ в долговременном состоянии, передайте его со значением `null` в `user_state_update`.
- Чтобы сбросить прогресс пользователя, передайте `null` для всех ключей, которые вы сохраняли, например:

```json
{
  "user_state_update": {
    "total_games": null,
    "total_correct": null,
    "best_score": null,
    "completed_quizzes": null,
    "level": null
  }
}
```

## 3. Настройки конкретного устройства

Используйте поле ответа `application_state_update`.

Это состояние возвращается в `state.application` и подходит для параметров, связанных с конкретным экземпляром приложения или устройством, а не с общим прогрессом пользователя:

```json
{
  "application_state_update": {
    "difficulty": "normal",
    "voice_hints": true,
    "sound_effects": false,
    "preferred_quiz_id": "capitals"
  }
}
```

Что хранить:

- `difficulty` - сложность на этом устройстве.
- `voice_hints` - включены ли голосовые подсказки.
- `sound_effects` - включены ли звуковые эффекты.
- `preferred_quiz_id` - последний выбранный режим или тема на этом устройстве.

Как очищать:

- Для удаления отдельной настройки передайте ключ со значением `null` в `application_state_update`.
- Для сброса настроек устройства передайте `null` для всех ключей настроек, которые использует навык.

## Лимиты и формат

Состояние хранится как JSON-объекты. Практическое правило: держите каждый объект состояния маленьким, не кладите туда тексты вопросов, большие списки, историю всех ответов или пользовательские данные, которые можно восстановить из вашей базы.

Ориентируйтесь на лимит около 10 КБ для передаваемого состояния. Поэтому вместо полных вопросов храните идентификаторы:

```json
{
  "session_state": {
    "asked_question_ids": ["q1", "q5", "q9"]
  }
}
```

а не:

```json
{
  "session_state": {
    "asked_questions": [
      {
        "id": "q1",
        "text": "Очень длинный текст вопроса...",
        "answers": ["..."]
      }
    ]
  }
}
```

## Пример ответа после правильного ответа

```json
{
  "response": {
    "text": "Верно. Следующий вопрос: столица Канады?",
    "end_session": false
  },
  "session_state": {
    "quiz_id": "capitals",
    "question_index": 5,
    "current_score": 4,
    "asked_question_ids": ["q1", "q5", "q9", "q12"],
    "last_question_id": "q12"
  },
  "user_state_update": {
    "total_correct": 127,
    "best_score": 9
  },
  "application_state_update": {
    "preferred_quiz_id": "capitals"
  },
  "version": "1.0"
}
```

## Ошибка, которую легко допустить

Частая ошибка - перепутать область видимости состояния.

Например, если хранить `current_score` в `user_state_update`, то счет текущей игры может случайно пережить завершение сессии и смешаться со следующей викториной. Если хранить настройки устройства в `user_state_update`, пользователь может получить неожиданную сложность или звук на другом устройстве. Если хранить общий прогресс в `session_state`, он исчезнет после окончания диалога.

Правило выбора:

- Временно и только до конца диалога - `session_state`.
- Долговременный прогресс пользователя - `user_state_update`.
- Настройки конкретного устройства или экземпляра приложения - `application_state_update`.
