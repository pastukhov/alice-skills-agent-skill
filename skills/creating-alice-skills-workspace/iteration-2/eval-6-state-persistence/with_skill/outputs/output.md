# Хранение состояния для викторины в навыке Алисы

Для викторины лучше разделить состояние на три независимых слоя: текущая попытка в рамках диалога, долгосрочный прогресс авторизованного пользователя и настройки конкретного устройства или приложения.

## 1. Счет текущей сессии

Используйте `session_state` в ответе навыка.

Что хранить:

```json
{
  "quiz": {
    "question_index": 3,
    "score": 2,
    "current_question_id": "capitals_003",
    "answered": ["capitals_001", "capitals_002"]
  }
}
```

Как читать: из `state.session` в следующем запросе.

Как писать: возвращать объект верхнего уровня `session_state` рядом с `response` и `version`.

```json
{
  "response": {
    "text": "Верно! Счет 2 из 3. Следующий вопрос...",
    "end_session": false
  },
  "session_state": {
    "quiz": {
      "question_index": 3,
      "score": 2,
      "current_question_id": "capitals_003",
      "answered": ["capitals_001", "capitals_002"]
    }
  },
  "version": "1.0"
}
```

Лимит: JSON-объект `session_state` максимум 1 КБ. Поэтому не храните полный текст вопросов, варианты ответов и длинную историю. Держите только идентификаторы и счетчики.

Очистка: чтобы сессионное состояние перестало храниться, не возвращайте `session_state` в ответе. Если сессия продолжается и состояние не изменилось, но его нужно сохранить для следующего хода, верните тот же `session_state`, который пришел в `state.session`.

## 2. Общий прогресс авторизованного пользователя между сессиями

Используйте `user_state_update` в ответе навыка.

Этот слой подходит для прогресса, который должен следовать за пользователем между устройствами, если пользователь авторизован через Яндекс ID. В запросе наличие `session.user.user_id` означает авторизованного пользователя; сохраненное состояние придет в `state.user`.

Что хранить:

```json
{
  "progress": {
    "total_games": 12,
    "best_score": 8,
    "completed_topics": ["capitals", "animals"],
    "level": 4
  }
}
```

Как писать:

```json
{
  "response": {
    "text": "Игра окончена. Ваш лучший результат теперь 8.",
    "end_session": false
  },
  "user_state_update": {
    "progress": {
      "total_games": 12,
      "best_score": 8,
      "completed_topics": ["capitals", "animals"],
      "level": 4
    }
  },
  "version": "1.0"
}
```

Лимит: JSON-объект `user_state_update` максимум 1 КБ.

Очистка: чтобы удалить поле из пользовательского состояния, отправьте это поле со значением `null`, например:

```json
{
  "user_state_update": {
    "progress": null
  }
}
```

Если пользователь не авторизован, не рассчитывайте на `state.user`: для долгосрочного персонального прогресса потребуется авторизация или собственное серверное хранилище.

## 3. Настройки конкретного устройства

Используйте `application_state` в ответе навыка.

Этот слой привязан к экземпляру приложения или устройства, то есть к `session.application.application_id`. Он подходит для настроек поверхности: озвучивать ли подсказки, предпочитаемая сложность на этом устройстве, режим коротких ответов, наличие экрана.

Что хранить:

```json
{
  "settings": {
    "difficulty": "normal",
    "voice_hints": true,
    "short_mode": false
  }
}
```

Как читать: из `state.application`.

Как писать:

```json
{
  "response": {
    "text": "Запомнила: на этом устройстве включен обычный уровень сложности.",
    "end_session": false
  },
  "application_state": {
    "settings": {
      "difficulty": "normal",
      "voice_hints": true,
      "short_mode": false
    }
  },
  "version": "1.0"
}
```

Очистка: чтобы очистить сохраненный стейт приложения, отправьте пустой объект:

```json
{
  "application_state": {}
}
```

Если настройки не менялись, `application_state` можно не возвращать: сохраненное состояние экземпляра приложения продолжит храниться.

## Итоговая схема

| Задача | Читать из запроса | Писать в ответе | Жизненный цикл |
| --- | --- | --- | --- |
| Счет текущей игры | `state.session` | `session_state` | Только до конца текущей сессии |
| Общий прогресс пользователя | `state.user` | `user_state_update` | Между сессиями для авторизованного пользователя |
| Настройки устройства | `state.application` | `application_state` | Между сессиями для конкретного экземпляра приложения |

Во всех ответах также должны быть обязательные поля протокола: `response.text`, `response.end_session` и `version: "1.0"`. Текст ответа ограничен 1024 символами, а каждый объект сохраняемого состояния для `session_state` и `user_state_update` ограничен 1 КБ.

## Легкая ошибка

Самая частая ошибка — обновить счет в памяти обработчика, но не вернуть `session_state` в следующем ответе. В Алисе сессионный стейт хранится только пока навык возвращает `session_state`; если поле пропущено, состояние перестанет приходить в следующих запросах. Поэтому для продолжающейся викторины возвращайте актуальный `session_state` на каждом ходе, даже если на этом ходе счет не изменился.
