# Правила ответа для карточки товара и кнопок в навыке Алисы

## 1. Всегда проектируйте ответ для двух типов поверхностей

Перед отправкой карточки проверьте, есть ли у пользователя экран:

```json
{
  "meta": {
    "interfaces": {
      "screen": {}
    }
  }
}
```

Если `meta.interfaces.screen` отсутствует, устройство не сможет показать изображение, список, галерею и ссылки. В этом случае ответ должен оставаться полноценным голосовым сценарием: опишите товар в `response.text` или `response.tts`, назовите цену и следующий доступный шаг словами.

Даже когда экран есть и вы отправляете `response.card`, поле `response.text` все равно обязательно. Оно используется как fallback, если карточка не отобразилась, и на голосовых устройствах будет основным ответом.

## 2. Выбирайте тип карточки по задаче

Используйте `response.card.type`:

- `BigImage` — один товар или главный товар дня: крупное изображение, заголовок, описание и кликабельная область.
- `ImageGallery` — несколько товаров в виде галереи изображений. В элементах указывайте `image_id`, `title` и при необходимости `button`.
- `ItemsList` — список товаров с описаниями. Подходит, когда нужно показать несколько позиций с названием, картинкой и коротким текстом.

Для `ImageGallery` и `ItemsList` обязательно передавайте непустой массив `items`. В справке по карточкам указаны такие ограничения: `ImageGallery` — от 1 до 10 изображений, `ItemsList` — от 1 до 5 изображений. Для каждого товара используйте заранее загруженный `image_id`; изображения загружаются в ресурсы навыка и имеют ограничения по размеру файла и общей квоте.

## 3. Делайте изображение и элементы списка кликабельными через `button`

Для клика по изображению товара добавляйте кнопку внутрь карточки:

```json
"card": {
  "type": "BigImage",
  "image_id": "1027858/example",
  "title": "Кофемолка Barista Mini",
  "description": "Компактная кофемолка для дома, 12 степеней помола.",
  "button": {
    "text": "Открыть товар",
    "url": "https://example.com/products/barista-mini",
    "payload": {
      "action": "open_product",
      "product_id": "barista-mini"
    }
  }
}
```

В `ImageGallery` и `ItemsList` аналогичная кнопка задается в каждом `items[].button`. У `ItemsList` также можно использовать `footer.button`; у `ImageGallery` footer не применяется.

## 4. Кнопки после ответа задавайте в `response.buttons`

Саджесты и ссылки под ответом передаются в `response.buttons`:

```json
"buttons": [
  {
    "title": "Купить",
    "payload": {
      "action": "buy",
      "product_id": "barista-mini"
    },
    "hide": true
  },
  {
    "title": "Открыть сайт",
    "url": "https://example.com/products/barista-mini",
    "payload": {
      "action": "open_url",
      "product_id": "barista-mini"
    },
    "hide": true
  }
]
```

Правила для кнопок:

- `title` обязателен, максимум 64 символа.
- `url` открывает страницу; ссылка ограничена 1024 байтами.
- `payload` должен быть JSON-объектом до 4096 байт.
- `hide: true` обычно лучше для одноразовых действий, чтобы пользователь не нажимал устаревшую кнопку.
- Если у кнопки нет `url`, по нажатию текст кнопки может прийти как реплика пользователя, поэтому не полагайтесь на видимый текст для бизнес-логики.

## 5. Обрабатывайте `ButtonPressed` только через `request.payload`

При клике по кнопке или кликабельному изображению навык получает запрос типа `ButtonPressed`, если в предыдущем ответе у нажатого элемента был непустой `payload`.

Маршрутизируйте действие так:

```js
if (request.type === "ButtonPressed") {
  const { action, product_id } = request.payload || {};

  if (action === "buy") {
    return buildBuyResponse(product_id);
  }

  if (action === "open_product") {
    return buildProductDetailsResponse(product_id);
  }
}
```

Нельзя при обработке `ButtonPressed` определять действие по видимому названию кнопки, `request.command`, `request.original_utterance` или тексту `title`/`text`. Эти строки относятся к интерфейсу: их могут изменить, локализовать, сократить или продублировать на разных кнопках. Стабильный идентификатор действия, товара, варианта и шага сценария должен быть только в `payload`.

## 6. Минимальная структура ответа с карточкой товара

```json
{
  "response": {
    "text": "Кофемолка Barista Mini: компактная модель для дома, 12 степеней помола. Могу рассказать подробнее или помочь купить.",
    "card": {
      "type": "BigImage",
      "image_id": "1027858/example",
      "title": "Кофемолка Barista Mini",
      "description": "Компактная кофемолка для дома, 12 степеней помола.",
      "button": {
        "text": "Подробнее",
        "payload": {
          "action": "product_details",
          "product_id": "barista-mini"
        }
      }
    },
    "buttons": [
      {
        "title": "Купить",
        "payload": {
          "action": "buy",
          "product_id": "barista-mini"
        },
        "hide": true
      },
      {
        "title": "Открыть сайт",
        "url": "https://example.com/products/barista-mini",
        "payload": {
          "action": "open_url",
          "product_id": "barista-mini"
        },
        "hide": true
      }
    ],
    "end_session": false
  },
  "version": "1.0"
}
```

## 7. Проверочный чеклист

- `response.text` есть всегда и понятен без картинки.
- Перед visual-only сценарием проверяется `meta.interfaces.screen`.
- Для одного товара выбран `BigImage`; для нескольких товаров выбран `ImageGallery` или `ItemsList`.
- В галерее и списке соблюдены ограничения на количество элементов и заполнены `image_id`.
- Кнопки и кликабельные изображения содержат структурный `payload`.
- В `payload` есть стабильные поля вроде `action`, `product_id`, `variant_id`, `step`.
- Ссылки передаются только в `url`; голосовой fallback объясняет, что делать без экрана.
- `ButtonPressed` маршрутизируется по `request.payload`, а не по видимому тексту кнопки.
- В ответе есть обязательные `response.end_session` и `version: "1.0"`.
