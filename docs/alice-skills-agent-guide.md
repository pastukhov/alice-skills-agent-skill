# Навыки Алисы: рабочая документация для ИИ-агентов

Документ собран из официальной документации Яндекс Диалогов по навыкам Алисы:
https://yandex.ru/dev/dialogs/alice/doc/ru/

Используйте этот файл как краткую инструкцию для проектирования, реализации, тестирования и публикации навыка. Полный локальный корпус страниц лежит в `docs/alice-skills-agent-docs.md`, исходные Markdown-страницы - в `docs/raw-md/`, список источников - в `docs/source-pages.md`.

## Быстрый маршрут агента

1. Определить сценарий навыка: цель, аудиторию, пользовательские задачи, ограничения тематики и поверхности запуска.
2. Проверить модерационные требования до написания кода: название, активационные имена, описание, бренд, иконка, приветствие, содержание.
3. Спроектировать диалог: первый запуск, интенты, ветвления, состояния, ошибки распознавания, повторные вопросы, выход из навыка.
4. Выбрать backend и развертывание: свой сервер, Yandex Cloud, Vercel, Azure, Google Cloud или AWS.
5. Реализовать webhook, который принимает POST-запрос Диалогов и возвращает JSON ответа протокола `1.0`.
6. Добавить хранение состояния, карточки, кнопки, изображения, звуки, авторизацию и аналитику только если они нужны сценарию.
7. Протестировать навык в консоли, проверить edge cases, отправить на модерацию, затем настроить мониторинг.

## Минимальный webhook-контракт

Диалоги отправляют POST-запрос на backend навыка. В теле запроса всегда ориентируйтесь на:

- `meta` - устройство, локаль, часовой пояс, доступные интерфейсы (`screen`, `account_linking`, `audio_player`).
- `request` - пользовательский ввод. Типы: `SimpleUtterance`, `ButtonPressed`, `Show.Pull`.
- `session` - идентификаторы сессии, сообщения, навыка, пользователя и приложения; `session.new` отмечает начало сессии.
- `state` - сохраненное состояние сессии, пользователя или экземпляра приложения.
- `version` - текущая версия протокола, `1.0`.

Минимальный ответ:

```json
{
  "response": {
    "text": "Здравствуйте! Чем помочь?",
    "end_session": false
  },
  "version": "1.0"
}
```

Обязательные поля ответа:

- `response.text` - текст для показа и озвучивания, максимум 1024 символа; может быть пустым, если задан `tts`.
- `response.end_session` - `false`, чтобы продолжить разговор, или `true`, чтобы завершить.
- `version` - `1.0`.

Источник: `docs/raw-md/request.md`, `docs/raw-md/response.md`.

## Обработка ввода

Для `SimpleUtterance` используйте:

- `request.command` - нормализованная команда пользователя.
- `request.original_utterance` - исходная фраза пользователя.
- `request.markup.dangerous_context` - признак потенциально чувствительного контекста.
- `request.nlu.tokens` - токены.
- `request.nlu.entities` - распознанные сущности.
- `request.nlu.intents` - интенты и слоты, если настроен NLU.

Для `ButtonPressed` обрабатывайте `request.payload`, который вы сами положили в кнопку или карточку прошлого ответа.

Для шоу Алисы используйте `Show.Pull` и отдельный протокол утреннего шоу.

Источники: `docs/raw-md/request-simpleutterance.md`, `docs/raw-md/request-buttonpressed.md`, `docs/raw-md/request-show-pull.md`, `docs/raw-md/morning-show-protocol.md`.

## Состояние

Включите хранилище в консоли навыка, если нужно помнить данные.

- `session_state` - состояние внутри текущей сессии, до 1 КБ. Чтобы оно не пропало, возвращайте его в каждом ответе, пока оно нужно.
- `user_state_update` - состояние авторизованного пользователя между сессиями, до 1 КБ. Чтобы удалить поле, отправьте его со значением `null`.
- `application_state` - состояние экземпляра приложения или устройства, разрез соответствует `session.application.application_id`.

Источник: `docs/raw-md/session-persistence.md`.

## Интерфейс ответа

Текст и речь:

- Используйте `response.text` как fallback для устройств без экрана и для случаев, когда карточка не отобразилась.
- Используйте `response.tts` для произношения, ударений, пауз, эффектов и звуков.
- Проверяйте лимит 1024 символа для `text` и `tts`; теги звуков в `tts` не учитываются в лимите.

Кнопки:

- Добавляйте `response.buttons[]` для саджестов и ссылок.
- `title` - текст кнопки.
- `payload` - произвольный JSON для следующего `ButtonPressed`.
- `url` - ссылка, если кнопка должна открыть страницу.
- `hide` - скрывать ли кнопку после нажатия.

Карточки:

- `BigImage` - одно изображение.
- `ItemsList` - список изображений, от 1 до 5.
- `ImageGallery` - галерея, от 1 до 7.

Источники: `docs/raw-md/response.md`, `docs/raw-md/buttons.md`, `docs/raw-md/response-card-bigimage.md`, `docs/raw-md/response-card-itemslist.md`, `docs/raw-md/response-card-imagegallery.md`.

## Авторизация

Используйте связку аккаунтов, когда навыку нужны данные из внешнего сервиса или персональная информация.

Порядок:

1. Проверить, поддерживает ли устройство `meta.interfaces.account_linking`.
2. Запустить авторизацию ответом с директивой `start_account_linking`.
3. Настроить OAuth-сервер и связку аккаунтов в консоли.
4. После авторизации читать токен из заголовка `Authorization`; если заголовки недоступны, токен также передается в `session.user.access_token`.
5. Не показывать конфиденциальные данные до проверки токена.

Источники: `docs/raw-md/auth-when-to-use.md`, `docs/raw-md/auth-how-it-works.md`, `docs/raw-md/auth-create-server.md`, `docs/raw-md/response-start-account-linking.md`.

## Требования перед публикацией

До отправки на модерацию проверьте:

- Название: обычно 2-5 слов; один бренд допускается при подтверждении прав.
- Активационные имена: дополнительные имена обычно 2-4 слова; учитывайте произношение, формы и транслитерацию.
- Описание: на русском языке, соответствует реальной функциональности.
- Контент: без мата, оскорблений, незаконного, опасного, вводящего в заблуждение или кликбейтного содержания.
- Бренды: права на использование должны быть подтверждены.
- Приветствие: объясняет, что умеет навык и как продолжить.
- Иконка и категория: соответствуют содержанию.

Официальные юридические требования имеют приоритет над справочным пересказом.

Источники: `docs/raw-md/requirements.md`, `docs/raw-md/moderation.md`, `docs/raw-md/publish-settings.md`, `docs/raw-md/publication.md`.

## Тестирование и мониторинг

Минимальный тестовый набор:

- Первый запуск: `session.new = true`, пустая команда, приветствие.
- Обычный запрос `SimpleUtterance`.
- Неизвестная команда и повторный вопрос.
- Нажатие каждой кнопки с `payload`.
- Устройство без `screen`, если вы используете карточки.
- Устройство без `account_linking`, если вы используете авторизацию.
- Завершение через `end_session: true`.
- Сохранение и очистка `session_state`, `user_state_update`, `application_state`.

После публикации подключайте AppMetrica и мониторинг ошибок, если навык публичный или бизнес-критичный.

Источники: `docs/raw-md/test.md`, `docs/raw-md/appmetrica.md`, `docs/raw-md/monitoring.md`, `docs/raw-md/health-check.md`.
